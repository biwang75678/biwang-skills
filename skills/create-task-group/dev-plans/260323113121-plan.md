# create-task-group - 修复空群 bug（user_id 自动发现失败时建空群）

## 需求描述

建群脚本 `create_chat.py` 通过 `discover_user_id()` 从 sessions.json 自动解析用户 open_id。当自动发现失败时（sessions.json 中无匹配的飞书私聊 session），脚本仅输出警告但继续执行，导致创建出无用户、无 bot 的空群。用户在飞书中完全看不到该群。

**复现路径**：agent cji2 的 sessions.json 中没有飞书私聊 session 记录 → `discover_user_id` 返回空字符串 → `create_group` 被调用时 `user_ids=[]`、`owner_id=""` → 飞书 API 创建了空群。

## 调研与决策

### 现状分析
- `create_chat.py` 无 `--user-id` 参数，完全依赖自动发现
- 自动发现失败时静默继续，无 fail-safe

### 方案
1. 新增 `--user-id` 参数作为兜底
2. 当自动发现失败且未指定 `--user-id` 时，报错退出而非建空群

### 决策
采用上述方案，改动最小且完全解决问题。
