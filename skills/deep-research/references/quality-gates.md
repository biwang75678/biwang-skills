# 质量门控标准

本文档定义了 deep-research 报告的质量标准和验证协议。

## Anti-Truncation 协议

**禁用词清单**（报告中绝对不允许出现）：
- "内容省略" / "此处略" / "待补充" / "待完善"
- "Content continues..." / "Due to length..."
- "此处省略" / "详见上文" / "略"
- "[sections X-Y...]" / "Additional citations..."

**执行方式**：每个报告节在模板中嵌入 HTML 注释：
```html
<!-- ANTI-TRUNCATION: 本节必须完整生成，禁止使用任何占位符 -->
```

## Anti-Fatigue 协议

防止 LLM 在长报告后半部分质量下降：

- 每大节 >= 3 段落
- 散文优先（>=80% 散文，<=20% 列表/表格）
- 零占位符
- 证据密度：每个主要断言同句引用 `[N]`

## 来源质量标准

| 模式 | 最低来源数 | 每断言来源数 |
|------|-----------|------------|
| quick | 5+ | 1+ |
| standard | 10+ | 2+ |
| deep | 15+ | 3+ |

**来源多样性要求**：
- 类型多样：学术 + 新闻 + 技术文档 + 行业报告
- 时间覆盖：不全是同一年份
- 立场平衡：包含正反两面观点

## Zero-Tolerance Bibliography

- 每个 `[N]` 引用必须在来源表中有对应条目
- 禁止范围引用（如 `[3-50]`）
- 禁止省略号（如 "Additional citations..."）
- 禁止占位符条目
- `validate-report` 会自动检测这些违规

## 写作标准

| 原则 | Bad | Good |
|------|-----|------|
| 精确 | "显著改善了效果" | "降低死亡率 23% (p<0.01)" |
| 具体 | "多项研究表明" | "5 项 RCT (n=1,847) 显示" |
| 经济 | "值得注意的是" | 直接陈述事实 |
| 直接 | "据报道称" | 陈述发现，附引用 |

## Anti-Hallucination 协议

区分三类内容：

1. **事实（FACTS）**：来自搜索来源，必须即时跟 `[N]` 引用
2. **综合（SYNTHESIS）**：你的分析和推断，用"这表明..."/"由此可见..."标记
3. **不确定（UNCERTAINTY）**：用"未找到相关来源"而非编造信息

## 验证循环

```bash
# 1. 运行结构验证
python3 $CLI validate-report research-report.md

# 2. 检查结果
# - valid: true → 通过
# - valid: false → 查看 issues 列表，修复后重跑

# 3. 最多 3 轮修复
# 如果 3 轮后仍有问题，向用户报告剩余问题
```

### 验证项目（9 项）

1. 引用编号连续性和唯一性
2. 引用编号与来源表对应
3. 无范围引用（`[N-M]`）
4. 无占位符文本
5. 无截断标记
6. 报告中 URL 均在已访问列表中
7. 来源数量达标
8. 每个聚类有内容
9. 摘要已替换（非占位符）
